### 学生列表接口
<img width="577" alt="image" src="https://user-images.githubusercontent.com/8150260/173975140-c637b647-7901-4017-ba88-6ced5d0062f6.png">

> 接口地址: 
- 请求方式 GET
- 开发状态: 文档中

> 请求参数
- pageNo: 当前页码
- pageSize: 每页数量
- schoolId: 学校ID
- 学生激活状态
  - 按状态过滤数据
- 家长激活状态
  - 按状态过滤数据
- classid: 班级id
  - 按班级id过滤数据
- 搜索关键词
  - 按关键词过滤学校身份下的 昵称 或 邮箱

> 接口返回数据
- [] 学生列表
  - id: 主键id 可用于登录
  - schoolId: 学校ID
  - *fristname: 名
  - *lastname: 姓
  - email: 邮箱
  - 生日
  - 头像
    + 默认使用系统默认头像
    + 学生登录后如果设置了个人头像，则更新学校下的头像
  - 班级id
  - 监护人
    - 名
    - 姓
    - *邮箱
    - phone
  - 学生激活状态
  - 家长激活状态
  - 最后登录时间

### 邮箱检查接口
> 接口地址: 
- 请求方式 GET
- 开发状态: 完成文档
- 接口说明: 检查该邮箱是否已经存在该学校学生数据里，***（不是查询是否注册账户）***

> 请求参数
- email: 学生邮箱
  - 学生存在的条件为 学校id + email 重复
- 家长邮箱
  - 家长存在的条件为 学校id + 家长email + 学生姓+名 重复

> 接口返回数据
- email: boolean 是否存在
- 家长邮箱: boolean 是否存在

### 学生批量上传
<img width="606" alt="image" src="https://user-images.githubusercontent.com/8150260/173975394-6266bd7f-4d3e-4362-9964-e0f55b0cd3b1.png">

> 接口地址: 
- 请求方式 POST
- 开发状态: 文档中
- 前端逻辑：
  - 浏览器端解析表格 (解析库: http://sheetjs.com)
  - 页面显示需要导入的数据
  - 正则检查邮件格式是否正确
    - 不正确，显示为 "Email invalid"
  - 批量检查家长和学生邮箱是否可以导入 [邮箱检查接口](api/学生管理.md?id=邮箱检查接口)
    - 已存在的邮箱, 显示为 "Duplicate"
  - 用户可以自由多选，或全选（邮件检查不通过的数据，不可选择）

> 接口返回数据
- [] 学生列表
  - *fristname: 名
  - *lastname: 姓
  - email: 邮箱
  - 生日
  - 班级id
    + 从班级入口进来的话，就绑定为进来的班级
  - 监护人
    - 名
    - 姓
    - *邮箱
    - phone

### 添加学生

> 接口地址: 
- 请求方式 GET
- 开发状态: 文档中
- 接口逻辑
  1. 分别查询家长和学生邮箱是否重复
    + 存在重复则返回错误信息 "Student profile already exists"
  2. 创建数据
  3. 发送邀请邮件给家长 和 学生(存在邮箱的话)

> 接口返回数据
- *fristname: 名
- *lastname: 姓
- email: 邮箱
- 生日
- 头像
- 班级id
  + 从班级入口进来的话，就绑定为进来的班级
- 监护人
  - 名
  - 姓
  - *邮箱
  - phone
